- 继电器位定义是否与板卡映射一致
- 超声波换算系数和超时逻辑是否统一
- 运动状态判定周期与采样周期是否匹配
- 参数页面锁定逻辑是否符合题面
- 统计显示是否要求前导 0
- 阈值判断是 `<` 还是 `<=` 是否一致

## 8. 常见错误档案（第十六届省赛复盘）

> 以下错误均在第十六届省赛真题评测中实际踩到，按"现象 → 根因代码位置 → 修正方向"归档。

---

### 8.1 继电器吸合/断开方向相反

| 项目 | 内容 |
|------|------|
| **现象** | 应吸合却断开，应断开却吸合，在相邻测试步骤出现互换 |
| **根因** | `RELAY_BIT` 位掩码与板卡实际继电器位不一致 |
| **代码位置** | `main.c` — `#define RELAY_BIT` |
| **修正方向** | 以官方资源包行为为唯一基准：`Uln_Ctrl(0x10)` 即吸合继电器，`RELAY_BIT = 0x10` |
| **坑点** | OCR 读原理图时可能与实际字节顺序不符，优先看官方参考实现，不要只看原理图网名 |

---

### 8.2 参数界面调参时继电器/LED 状态丢失

| 项目 | 内容 |
|------|------|
| **现象** | 进入参数界面调参过程中，继电器意外断开；L8 该闪烁的变成常亮或熄灭 |
| **根因** | `output_lock` 分支直接冻结了全部 LED，L8 未继续执行闪烁逻辑 |
| **代码位置** | `main.c` — `void output_update()` → `if(output_lock)` 分支 |
| **修正方向** | 锁定时将 L1–L7 冻结，L8 仍按进入参数页时的 `move_state` 继续 STOP/WALK/RUN 逻辑 |

---

### 8.3 统计界面 NC 显示前导零格式错误

| 项目 | 内容 |
|------|------|
| **现象** | 期望显示 `NC 3`，实际显示 `NC 0003` |
| **根因** | `stat_display()` 固定输出 4 位数，高位补零 |
| **代码位置** | `main.c` — `void stat_display()` |
| **修正方向** | 当 `n < 1000/100/10` 时，对应位熄灭（`digital_tube(pos, 0xFF)`），不补 0 |
| **坑点** | 数码管段码 `0xFF` 表示全段熄灭，可用于"不显示某位" |

---

### 8.4 运动状态判断偏差（L 状态等级错误，距离显示偏移）

| 项目 | 内容 |
|------|------|
| **现象** | 应判 WALK 却显示 STOP；距离变化与状态切换对不上 |
| **根因一** | 距离采样周期与运动判定周期不统一，位移差取的是 200ms 差值，但阈值是按 1s 尺度设计的 |
| **根因二** | 状态锁定每次都触发，导致长时间无法更新状态 |
| **代码位置** | `main.c` — `void update_move_state()` / 主循环采样间隔 |
| **修正方向** | 确保 delta 比较使用同一时间窗；只在状态发生变化时才触发锁定 |

---

### 8.5 超声波距离偏差

| 项目 | 内容 |
|------|------|
| **现象** | 测距结果比期望值恒定偏大约 1.18%；远距离/弱回波时偶尔误返 0 |
| **根因一** | 换算系数 `0.0172` 与官方 `0.017` 不一致 |
| **根因二** | 回波超时用 `CH < 0x40` 而非 PCA 溢出标志 `CF`，导致远距离提前退出 |
| **代码位置** | `superwave.c` — 换算系数 / 回波等待条件 |
| **修正方向** | 系数改为 `0.017`；超时改为 `while(RX && !CF)` + `if(CF)` 判断 |

---

### 8.6 接近判定边界异常

| 项目 | 内容 |
|------|------|
| **现象** | 距离为 0 时不触发接近，某些场景错误触发 |
| **根因** | `distance > 0 && distance < dis_para` 的组合条件与评测基准不一致 |
| **代码位置** | `main.c` — `void judge_state()` |
| **修正方向** | 以官方参考行为为准，使用纯 `distance < dis_para` |

---

### 8.7 温度采样滞后一拍

| 项目 | 内容 |
|------|------|
| **现象** | 场景切换时温度判定比实际值落后一个周期，相邻测试步骤结果像"串了一步" |
| **根因** | `read_temp_data()` 里"先 convertT 再 readT"实际读的是上一次转换结果 |
| **代码位置** | `main.c` — `void read_temp_data()` |
| **修正方向** | 改为"先 readT，再 convertT"，每次调用都能拿到当前最新值 |

---

### 8.8 Git 仓库状态混乱导致版本错误

| 项目 | 内容 |
|------|------|
| **现象** | 提交列表全部变黄；以为在某版，实际不在该分支 |
| **根因** | `git switch --detach` 进入 detached HEAD 状态；另有未完成的 cherry-pick 残留 |
| **修正方向** | 回到指定版本时应 `git switch main` + `git reset --hard <commit>`，不在 detached HEAD 下直接改代码 |
